What is this?

This tool is used to create an HTTPS upload endpoint for customers. The endpoint can be used to upload file(s) via a web browser or command line tools, such as curl.

This can be thought of as an alternative to the existing MongoDB dropbox tool.

What do I need to do?

Provide the form with the necessary parameters, most importantly the customer name / identifier. The tool will then create an endpoint for the customer to access, sign it, and ship it off to and s3 bucket.

You provide the customer with a link. They can then upload via their web browser or curl. Traffic is HTTPS and files are stored in S3 with encryption at rest.

How do I access what they upload?

All files are stored encrypted-at-rest in an s3 bucket on Amazon. Those files can be viewed via this application as well. When you download a file thorugh this site, the application generates a signed temporary download URL. The file is then downloaded directly from s3, thereby reducing the need for our servers to transfer and store the file(s). It's also quicker.



Parameters

Customer Name / Identifier

Enter an identifying string here, such as a customer name or CS ticket number. The file(s) uploaded by the customer will be placed in a folder of this name. You can only use Alphanum characters, numbers, - and _ in this field.

Expiration Date

The form this tool generates will eventually expire. The expiration date should be entered here. Try to keep the value low, but long enough to be of use to your customer. This defaults to 2 weeks past the current date.

If a form expires, you can always generate a new one.
